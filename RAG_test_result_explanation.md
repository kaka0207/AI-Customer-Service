# RAG 测试结果如何得来及其意义

这个项目的测试结果，主要来自一套离线 RAG 检索评测。它评估的不是最终回答写得好不好，而是评估用户提出问题后，RAG 能不能从知识库里检索到正确、相关、足够有用的资料。也就是说，它重点测试的是召回质量。

## 一、测试数据怎么来的

项目里预先准备了一组典型客服问题，一共 12 条，覆盖维护保养、故障排查、场景适配、选购咨询、语义改写等场景。

每个测试问题都人工标注了两类标准答案依据：期望关键词和期望资料来源。

```text
示例问题：扫地机器人吸力变弱，清理不干净怎么办？
期望关键词：主刷、边刷、毛发、尘盒、滤网、吸力
期望来源：扫地机器人100问2.txt、故障排除.txt
```

这些标注相当于测试集里的参考标准。系统检索出来的内容越能覆盖这些关键词和来源，说明检索质量越好。

## 二、测试过程怎么跑

测试时，会分别使用不同检索模式对同一批问题进行检索，例如 BM25 关键词检索、向量检索和 Hybrid RRF 混合检索。每个问题都会取 Top-K 条检索结果，之前的评测通常看 Top-3，也就是每个问题最相关的前三段资料。

```text
读取测试问题
-> 用某种检索模式检索知识库
-> 得到 Top-K 文档片段
-> 拼接这些片段的正文
-> 检查是否命中期望关键词
-> 检查是否命中期望来源
-> 汇总每个问题和每种检索模式的指标
```

## 三、指标怎么算

第一个指标是平均关键词召回率。它看的是期望关键词里，有多少出现在检索结果中。

```text
关键词召回率 = 命中的关键词数量 / 期望关键词总数
例如：5 / 6 = 83.33%
```

所有问题的关键词召回率再求平均，就得到某种检索模式的平均关键词召回率。

第二个指标是来源命中率。它看的是检索出来的资料，是否来自预期的知识文档。如果 Top-K 结果里出现了任意一个期望来源，就算该问题来源命中。

```text
来源命中率 = 来源命中的问题数 / 总问题数
例如：11 / 12 = 91.67%
```

第三个指标是全关键词命中率。这个指标更严格，要求一个问题的所有期望关键词都被检索结果命中。

```text
全关键词命中率 = 所有关键词都命中的问题数 / 总问题数
例如：7 / 12 = 58.33%
```

## 四、已有测试结果如何理解

```text
BM25：
平均关键词召回率 80.28%
来源命中率 83.33%
全关键词命中率 66.67%

向量检索：
平均关键词召回率 82.22%
来源命中率 83.33%
全关键词命中率 58.33%

Hybrid RRF 混合检索：
平均关键词召回率 81.67%
来源命中率 91.67%
全关键词命中率 58.33%
```

这些结果说明三种检索方式各有特点。BM25 的全关键词命中率最高，说明它对明确关键词、专业术语、耗材名称这类问题比较稳，例如 HEPA 滤网、滤网堵塞、3-6个月。

向量检索的平均关键词召回率略高，说明它对语义相似问题更友好。用户不一定照着知识库原文提问，可能会说地上还有灰、感觉吸不动东西，这类表达更适合向量检索。

Hybrid RRF 的来源命中率最高，达到 91.67%，说明混合检索更容易把正确资料来源召回出来。它同时利用了向量检索的语义能力和 BM25 的关键词能力，所以整体更稳。

## 五、这个测试结果有什么意义

- 第一，它能量化 RAG 的检索质量。如果没有评测，只能凭感觉判断回答好像不错；有了指标，就能知道检索是否真的找到正确资料。
- 第二，它能比较不同检索策略。BM25、向量检索、混合检索到底谁更适合当前知识库，可以根据评测结果决定，而不是靠猜。
- 第三，它能发现薄弱场景。故障排查类通常表现较好，因为关键词明确；语义改写类更难，因为用户表达更口语化。
- 第四，它能指导参数调优。Top-K、chunk 大小、是否开启 rerank、向量和 BM25 权重，都可以通过评测结果来调整。
- 第五，它能防止后续改动造成退化。以后修改知识库、切分策略、Embedding 模型、检索模式或 rerank 配置后，都可以重新跑评测做回归验证。

## 六、需要注意的局限

这套测试主要评估检索结果是否命中资料，不等同于完整评估最终大模型回答质量。它能说明模型拿到的参考资料质量如何，但不能完全证明最终回答一定优秀。

另外，关键词匹配比较机械。如果检索结果里出现了同义表达，但没有出现完全一样的关键词，指标可能不会算命中。因此它适合做工程上的稳定评估，但不能完全替代人工评价或大模型评审。

当前项目已经切换成混合检索并开启 rerank。之前这些结果主要来自已有离线评测；如果要得到当前配置下的最新结果，需要重新跑一次评测，才能判断 rerank 开启后指标是否进一步提升。

## 面试总结表达

这个项目的 RAG 测试结果是通过离线评测集计算出来的。每个测试问题都人工标注了期望关键词和期望资料来源，系统分别用 BM25、向量检索和 Hybrid RRF 对知识库做 Top-K 检索，然后统计关键词召回率、来源命中率和全关键词命中率。这个测试的意义是量化 RAG 的召回质量，比较不同检索策略，发现薄弱场景，并为后续参数调优和回归测试提供依据。当前它主要评估检索阶段，不等同于完整的最终回答质量评测。
